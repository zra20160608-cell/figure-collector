**启动日期**: 2026-03-27  
**项目经理**: PM-Agent  
**状态**: 🟡 需求调研阶段  

---

## 1. 项目概述

### 1.1 项目背景
用户（尧）需要一个专注于国内手办市场的信息聚合平台。目前国内手办信息分散在多个平台（B站会员购、淘宝、京东、AC模玩网等），用户需要一个统一入口来查询、监控和获取上新提醒。

### 1.2 核心价值主张
- **信息聚合**: 一站式查询多个平台的手办信息
- **价格透明**: 价格历史追踪，避免买贵
- **上新提醒**: 不错过心仪手办预售
- **社区交流**: 手办爱好者交流心得

### 1.3 目标用户
| 用户类型 | 特征 | 核心需求 |
|----------|------|----------|
| 手办爱好者 | 定期购买，关注新品 | 上新提醒、价格监控 |
| 收藏者 | 追求稀有/限定款 | 稀缺性信息、二级市场价 |
| 新人 | 刚入坑，不了解市场 | 入门指南、价格参考 |
| 二手交易者 | 关注涨跌，低买高卖 | 价格趋势、市场热度 |

## 2. 初步需求分析

### 2.1 数据源需求
- **B站会员购**: 国内最大手办预售平台
- **淘宝**: 各类手办店铺（模玩熊、猫受屋等）
- **京东**: 自营+第三方手办
- **AC模玩网**: 专业手办资讯
- **Hpoi**: 手办数据库（参考）

### 2.2 核心功能模块
```
┌─────────────────────────────────────────────────────────┐
│                    手办信息收集网站                        │
├──────────────┬──────────────┬──────────────┬─────────────┤
│   数据抓取    │   展示查询    │   用户功能    │   社区功能   │
├──────────────┼──────────────┼──────────────┼─────────────┤
│ - 多平台爬虫  │ - 手办列表    │ - 价格提醒    │ - 评论     │
│ - 数据清洗    │ - 详情页面    │ - 上新通知    │ - 收藏     │
│ - 定时更新    │ - 搜索筛选    │ - 个人中心    │ - 分享     │
│              │ - 价格历史    │ - 订阅管理    │ - 讨论区   │
└──────────────┴──────────────┴──────────────┴─────────────┘
```

### 2.3 待细化功能点
1. **信息抓取**: 爬虫策略、反爬应对、数据更新频率
2. **价格监控**: 监控周期、提醒阈值、通知方式
3. **上新提醒**: 推送渠道（邮件/短信/微信/APP推送）
4. **社区功能**: UGC内容、评分系统、话题讨论

## 3. 建议的技术方案

### 3.1 技术栈建议
| 层级 | 推荐技术 | 备选方案 |
|------|----------|----------|
| 前端 | Next.js + Tailwind | Vue3 + Element Plus |
| 后端 | Python FastAPI | Node.js NestJS |
| 数据库 | PostgreSQL + Redis | MySQL + Redis |
| 爬虫 | Scrapy + Playwright | requests + bs4 |
| 部署 | Docker + Nginx | Vercel + Railway |

### 3.2 系统架构简图
```
用户 → CDN → Nginx → Next.js 前端
              ↓
         FastAPI 后端
              ↓
    ┌─────────┴─────────┐
PostgreSQL      Redis    爬虫服务
```

### 3.3 关键技术挑战
1. **反爬虫**: 各平台反爬策略，需要代理池和验证码处理
2. **数据一致性**: 多源数据整合，去重和标准化
3. **实时性**: 上新信息需要及时同步
4. **扩展性**: 未来支持更多平台和功能

## 5. 风险评估

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 数据源不稳定 | 高 | 高 | 多平台备份、数据缓存、用户反馈渠道 |
